# Task 01 - Architect

## Goal
Assess the repository and define the smallest complete execution order for the assignment.

## Inputs
Read:
- `CLAUDE.md`
- `docs/architecture.md`
- `docs/delivery-scope.md`
- `docs/ai-workflow.md`
- `docs/implementation-plan.md`

Inspect the repository structure and currently existing files.

## Required behavior
Return:
1. a concise execution order in 6-10 steps
2. any setup gaps or missing dependencies
3. any risks of scope creep
4. a recommendation for the smallest complete version of the assignment

## Constraints
- do not modify files
- do not generate code
- do not expand the scope
- keep recommendations concrete and minimal

## Required output format
Use these exact sections:
- Execution order
- Setup gaps
- Scope risks
- Smallest complete version
